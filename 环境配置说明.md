## 二、环境准备（从0配置步骤）

### 第1步：安装Python 3.13

确保已安装 Python 3.13 或兼容版本，并将 `python` 和 `pip` 加入系统PATH。

### 第2步：创建虚拟环境（推荐）

```bash
cd D:\code\smart_search_project
python -m venv .venv
# Windows激活
.venv\Scripts\activate
# Linux/Mac激活
# source .venv/bin/activate
```

### 第3步：安装依赖

项目提供两个依赖清单：

**一键安装（推荐）**:
```bash
pip install -r requirements_current.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**最简安装**（如需最小集，见 requirements.txt）:
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 7.1 Hugging Face国内镜像

项目已在以下位置配置国内镜像：

```python
# manage.py 第24行
os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"

# 250/250.py 第14行

```

### 7.2 SSL证书处理

由于国内网络环境有时会遇到SSL问题，项目在 `manage.py` 中强制跳过了SSL证书验证：

```python
ssl._create_default_https_context = ssl._create_unverified_context
```

### 7.3 代理清理

`manage.py` 和 `settings.py` 都会主动清理HTTP代理环境变量，确保AI API调用不被代理干扰：

```python
os.environ.pop('http_proxy', None)
os.environ.pop('https_proxy', None)
```

### 7.4 代理排除配置

针对需要使用代理的场景，已在 `.env` 配置排除列表：

```
NO_PROXY=ark.cn-beijing.volces.com,localhost,127.0.0.1
```

## 九、常见问题与排查

### 9.1 MySQL连接失败

**现象**: `django.db.utils.OperationalError: (1045, ...)`
**排查**:
1. 确认MySQL服务已启动: `net start mysql`
2. 确认 `.env` 中 `DB_PASSWORD` 正确
3. 确认密码加密方式兼容: `ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '密码';`

### 9.2 cryptography 报错

**现象**: `Authentication plugin caching_sha2_password cannot be loaded`
**解决**:
```bash
pip install cryptography
```

### 9.3 FAISS索引文件损坏

**现象**: 搜索无结果或报IndexError
**解决**: 删除 `data/faiss_index.bin` 和 `data/faiss_index.bin.meta`，然后重新构建：
```bash
python -c "from search_engine.vector_index import initialize_index; initialize_index(force_rebuild=True)"
```

### 9.4 AI API调用失败

**现象**: 日志中出现 `Volcengine Call failed`
**排查**:
1. 检查API密钥是否过期（火山引擎ARK控制台确认）
2. 检查网络是否能访问 `ark.cn-beijing.volces.com`
3. 检查 `.env` 中 `NO_PROXY` 配置是否包含该域名

### 9.5 向量模型下载失败

**现象**: `sentence-transformers` 加载模型时超时
**解决**:
1. 确认 `HF_ENDPOINT=https://hf-mirror.com` 已设置
2. 手动下载: 从国内镜像手动下载模型文件放到缓存目录
3. 或者使用代理: 临时设置 `https_proxy` 环境变量

### 9.6 250.py PDF提取异常

**现象**: 处理PDF时卡住或报错
**排查**:
1. 检查PDF是否为扫描件（250.py自动跳过文本不足100字符的PDF）
2. 检查火山引擎API密钥是否有效（250.py使用独立密钥）
3. 确认 `D:\code\pdfs_to_process` 目录存在

### 9.7 PyTorch CPU vs GPU

当前环境安装的是CPU版PyTorch（`torch 2.10.0+cpu`），向量推理使用CPU。
如果服务器有NVIDIA GPU且安装了CUDA，可以重装GPU版以加速向量搜索：
```bash
pip uninstall torch
pip install torch --index-url https://download.pytorch.org/whl/cu124
```
