# 前端推荐报告模块剥离 — 工作报告

**日期：** 2026-05-03
**分支：** main

---

## 一、任务背景

原项目 `smart_search_project` 是一个纯 Django 后端项目，前端展示仅有单个服务端渲染的 Django 模板（`search_engine/templates/search.html`），通过 Tailwind CSS CDN 和 FontAwesome CDN 实现样式，无任何前端框架、构建工具或 package.json。

本次任务目标：将前端展示部分剥离为独立模块，前端不再直接调用后端接口，改为读取本地 JSON 文件渲染页面，同时保留未来重新接入真实 API 的能力。

## 二、实现方案

**技术选型：** 原生 JavaScript ES Modules，无需构建工具，复用项目现有的 Tailwind CSS CDN + FontAwesome CDN 方案。

**架构设计：**
- 组件类使用静态 `render()` 方法返回 DOM 元素
- 单向数据流：`DataProvider.fetchData()` → `parseRecommendationData()` → `Report.render()` → DOM mount
- DataProvider 抽象数据访问层，未来切换 API 仅需修改一处配置

## 四、功能要点

### 4.1 数据解析（recommendationParser.js）

| 功能 | 实现 |
|------|------|
| `final_recommendation` 类型兼容 | 支持 JSON 字符串和已解析数组两种输入 |
| 解析失败处理 | `console.error` 输出错误；若 `search_results` 存在则降级渲染并显示警告横幅 |
| 产品名匹配 | `trim().toLowerCase()` 规范化，大小写不敏感 |
| 字段优先级（DOI） | `final_recommendation.doi` → `search_results.metadata.source_doi` |
| 字段优先级（Similarity） | `final_recommendation.similarity_score` → `search_results.similarity` |
| DOI 清洗 | 去除空白，拒绝空字符串 / `"N/A"` / `"nan"` |
| 排序 | 保持 `final_recommendation` 原始顺序 |

### 4.2 页面展示

**标题：** Biochemical Product Recommendation Report

**顶部信息卡片（2×2/4×1 响应式网格）：**
- User Input ← `formatter.raw_input`
- Refined Input ← `formatter.refined_input`
- Verification Score ← `formatter.verification_score`（颜色标识：绿≥80/黄≥50/红<50）
- Iteration Count ← `formatter.iteration_count`

**推荐卡片字段：**
| 字段 | 数据来源 | 展示方式 |
|------|----------|----------|
| Product Name | `final_recommendation.product_name` | 标题 |
| Evaluation Score | `final_recommendation.score` | 进度条 + 数值，缺失显示 N/A |
| Semantic Similarity | 4 位小数 | 行内文本 |
| Recommendation Reason | `final_recommendation.reason` | 蓝色信息框 |
| Functional Description | `search_results.description` | `white-space: pre-line` 保留换行 |
| DOI | DOI 链接 | `https://doi.org/{doi}`，`target="_blank" rel="noopener noreferrer"` |
| IUPAC Name | `metadata.iupac_name` | 标签行 |
| Tags | `metadata.tags` | 灰色标签列表 |

**Top Recommendation：** 首条结果金色边框高亮 + 🏆 徽章

### 4.3 安全措施

- 所有来自 JSON 的动态文本使用 `textContent` 设置，不使用 `innerHTML`
- `innerHTML` 仅用于 JS 文件中硬编码的静态结构标记
- DOI 链接仅在值有效时创建
- 中文支持：`<meta charset="UTF-8">`

### 4.4 错误处理

| 层级 | 场景 | 行为 |
|------|------|------|
| 数据获取 | 网络错误/404/JSON格式错误 | 全屏错误界面 + 重试按钮 |
| 数据解析 | `final_recommendation` 解析失败 | 降级至 `search_results` 卡片 + 警告横幅 |
| 渲染 | 字段缺失 | 该区域静默省略；两者皆空显示 "No recommendations found" |

## 七、后续接入 API 指引

修改 `frontend/js/app.js` 中 DataProvider 构造参数：

```js
// 当前（本地 JSON）
var dataProvider = new DataProvider({
  dataSource: 'local',
  localPath: './data/interfaceData.json'
});

// 接入 API 后
var dataProvider = new DataProvider({
  dataSource: 'api',
  apiEndpoint: '/api/recommendation/',
  apiMethod: 'POST',
  timeout: 15000
});
```

如果 API 返回的 JSON 结构与 `interfaceData.json` 一致（根级 `formatter` 对象），无需其他代码改动。详见 `frontend/README.md`。
